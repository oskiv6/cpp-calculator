# C++ Calculator

Mathematical expression evaluator featuring a custom hand-written Lexer, Recursive Descent Parser, and Tree-Walking Interpreter.

## Features

- **Arithmetic Operations:** Supports `+`, `-`, `*`, `/`, and `%` (modulus).
- **Operator Precedence:** Correctly handles standard mathematical order of operations.
- **Parentheses:** Support for grouping expressions to override precedence (e.g., `(1 + 2) * 3`).
- **Floating Point:** Full support for decimal numbers and precise division.
- **Variables:** Define and reuse variables within a session (e.g., `x = 10.5`, `x * 2`).
- **Multiple Modes:**
  - **REPL:** Interactive terminal-based calculator.
  - **File Execution:** Run a batch of expressions from a text file.
  - **Debug Mode:** Visualize the Abstract Syntax Tree (AST) generated by the parser.

## Supported Platforms

The project is designed to be cross-platform and has been tested on:
- **macOS** (Clang)
- **linux** (Clang)

## Building the Program

The project uses a custom build system implemented in `nob.c`. It automatically detects available compilers (`clang++`, `g++`, or `c++`).

### Prerequisites
- A C compiler (to build the build-system itself).
- A C++17 compatible compiler.

### Build Instructions
```bash
# 1. Compile the build system
cc -o nob nob.c

# 2. Run the build system to compile the project
./nob

# The executable will be generated at ./build/calc
```

## Usage Examples

### Interactive REPL
Launch the interactive mode by running the executable without arguments:
```bash
./build/calc 
# or ./nob run
> a = 10
Result: 10
> (a + 5) * 2
Result: 30
```

### Running a File
Execute a script containing multiple expressions:
```bash
# Create a test file
echo "x = 3.14" > script.txt
echo "radius = 10" >> script.txt
echo "area = x * radius * radius" >> script.txt

# Run the file
./build/calc run script.txt
```

### Debug Mode
Use the `-d` flag to see the AST structure for each expression:
```bash
./build/calc -d
> a = 1 + 2 * 3
--- AST Structure ---
Assignment (a):
  BinaryExpr (+):
    Literal: 1
    BinaryExpr (*):
      Literal: 2
      Literal: 3
---------------------
Result: 7
```

## Project Structure
- `src/core/`: Lexer, Parser, and AST definitions.
- `src/interpreter/`: Evaluation logic and variable state management.
- `src/util/`: Common utilities and data structures.
- `nob.c`: The automated build script.
